# GitHub Recon for Bug Bounty

## **Objectives**

1. Organization Recon – Finding sensitive data related to an organization.
2. Employee Profiles – Identifying developer accounts, leaks, and insider information.

## **What You Can Find During Recon**

* Source Code Leakage
* Passwords
* Tokens
* API Keys
* Usernames
* Sensitive Files
* Internal IPs
* AWS Secrets
* SSH, FTP Passwords

\##------------------------------------------------------------------------------ **Automation**-------------------------------------------------------------------------------

1.Using GitHub Dork Helper

	* Visit: [https://vsec7.github.io](https://vsec7.github.io)

2.Gitleaks

	# Git commits + branches
	gitleaks detect --source=<cloned-repo> --log-opts="--all"  --report-path history.json


	# Untracked + staged files
	gitleaks detect --source=<cloned-repo> --no-git --redact --report-path working.json

3.Trufflehog:
sudo docker run --rm -it -v "$PWD:/pwd" trufflesecurity/trufflehog:latest github --repo https://github.com/trufflesecurity/test_keys --issue-comments --pr-comments --results=verified,unknown >>truffle_repo.txt
sudo docker run --rm -it -v "$PWD:/pwd" trufflesecurity/trufflehog:latest github --org=trufflesecurity --issue-comments --pr-comments --results=verified,unknown >>truffle_org.txt
